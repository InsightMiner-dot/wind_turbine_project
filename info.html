Training Configuration Guide

1. Learning Rate

The learning rate controls how much the model weights change with each step. A good starting point is 0.001. Try smaller values (e.g., 0.0001) if the loss fluctuates or diverges.

2. Number of Epochs

The number of full training cycles through the dataset. Use 5-20 for small datasets. Use early stopping if the validation loss plateaus.

3. Batch Size

Number of samples processed before the model updates. Common values: 4, 8, 16. Larger batch size → faster training, but needs more memory.

4. Model Architectures

  • EfficientNet-B0: Balanced for accuracy and performance. Use when accuracy matters and you have a GPU.
  • ResNet-18: Simpler, fast, and often good enough. Use on CPU or smaller datasets.

5. Optimizers

  • Adam: Good default. Adapts learning rate per parameter.
  • AdamW: Like Adam but with better regularization (less overfitting). Use with weight decay.
